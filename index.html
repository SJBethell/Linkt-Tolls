Step 03 — Configure

Rules & hours

Classification is driven primarily by Route rules (deterministic — same toll gantry pair always classifies the same way). Working hours and commute windows are the fallback for trips that don't match a known route.

Route rules

Toll-route classification

Every Linkt trip has a fixed description (e.g. Kedron to Bowen Hills). This table classifies each known route as one of three classes:

  • Personal (commute corridor) — always personal. Use for the driver's home ↔ office drive (e.g. Kedron to Bowen Hills).
  • Work (site / tunnel) — always work. Use for specific client / site / tunnel routes that are only driven for work (e.g. Legacy Way West).
  • Commute (window-aware) — personal during the AM/PM commute windows below and on weekends/public holidays; work outside those windows. Use for motorway-through toll points that drivers pass through on both the daily commute and mid-day job runs (e.g. Murarrie North, Kuraby South).

Route rules take priority over working hours. A 07:45 Gateway Motorway toll on Murarrie North now classifies as PERSONAL (commute window), while an 11:20 Gateway Motorway toll on the same route classifies as WORK.

Trip description (contains, case-insensitive) Class Note Actions

Matching is case-insensitive substring — "Kedron to Bowen Hills" matches any Linkt row whose description contains that phrase. Add new routes as they appear in statements. The first matching row wins, so put more specific patterns above general ones.

Working hours (fallback only)

Your week

Used only when a trip does not match any route rule. Inside these hours on a working day = WORK. Outside = PERSONAL. Default is Mon–Fri 06:00–19:00 to cover typical tradie shifts.

Route rules above now handle commute detection. This legacy setting is kept for edge cases.

How it works

Exact rules, amounts, and decision logic — no generics. Everything below reflects what the code actually does.

Classification logic

Decision order for every trip

Each trip is tested through the following steps in order. The first rule that matches wins — the rest are skipped.

1
Manual override

If you have ticked Force work or Force personal in the Override column, that choice is used unconditionally. Nothing else runs.

→ WORK → PERSONAL
2
Driver default rule

Each driver in the Drivers & Tags table has an optional Default rule field. If it is set to always-work or always-personal, every toll on that tag is classified that way regardless of time, day, or route. Currently configured:

Always work: Addison Mitchell (980KJ7), Bailee Reed-Sgro (478A4C)
All others: Auto — continues to steps below
3
Route rules highest priority

The Linkt trip description (e.g. Gateway Motorway — Murarrie North) is matched case-insensitively against the route table in Settings. The first matching row wins. Three possible outcomes:

CLASS: work — always WORK
Route is specific to a job site or tunnel — always billable to the company. Examples: Legacy Way West, Bowen Hills to Toombul.
CLASS: personal — always PERSONAL
Route is the driver's home ↔ office commute corridor — always personal. Examples: Kedron to Bowen Hills, Bowen Hills to Kedron.
CLASS: commute — window-aware (PERSONAL in peaks, WORK mid-day)
Used for motorway through-tolls (Gateway Murarrie, Logan Kuraby) that are hit on both the daily commute and legitimate mid-day job runs. The same gantry can be work or personal depending on time of day.
PERSONAL Weekend or public holiday (Saturday or Sunday as configured)
PERSONAL Weekday, but time falls inside AM window 06:00–09:30 or PM window 15:00–18:30
WORK Weekday, outside both commute windows — assumed mid-day job run
Current routes in this class: Murarrie North, Murarrie South, Kuraby North, Kuraby South. You can adjust the windows in Settings → Working hours.

If no route rule matches, classification continues to step 4.

4
Working day & hours fallback

Only reached if no route rule matched. Checks the working calendar in Settings:

PERSONAL Day is Saturday or Sunday (not marked as a working day)
PERSONAL Weekday but time is before 06:00 or after 19:00 (the configured working hours)
WORK Weekday, time between 06:00 and 19:00 — no other rule matched
✓
The Rule column in the trip table shows exactly which step fired for every row, so you can see the reasoning at a glance.

Amounts & repricing

Linkt amount vs charge amount

Two dollar figures appear in the trip table. They are different things.

Linkt Amt
What Linkt charged

The actual toll deducted from the Linkt account. Linkt charges vehicle class rates — your vehicles are registered as Light Commercial Vehicle (LCV), so the LCV rate applies. Examples from your statement:

Gateway Murarrie (LCV)$9.30
Clem7 (LCV)$10.37
AirportlinkM7 (LCV)$8.67–$11.36
Logan Kuraby (LCV)$5.74
Legacy Way (LCV)$11.13
Charge (CAR rate)
What to invoice the driver

For personal trips, you can reprice to the CAR (Class 2) rate — the lower rate a private car would pay. This is more defensible for FBT / employee reimbursement. Click → CAR on a row, or Reprice personal as CAR to do all at once.

Gateway Murarrie (CAR)$5.78
Clem7 (CAR)$6.50
AirportlinkM7 (CAR)$7.16
Logan Kuraby (CAR)$3.61
Legacy Way (CAR)$7.00

The Charge column shows a dash for work trips (not invoiced). For personal trips it shows the Linkt amount unless a CAR rate override is applied, in which case the CAR rate is shown in amber. The Xero export always uses the Charge column, not the Linkt amount.

KPI cards

What the four numbers mean
Total trips

Count of all parsed rows across all drivers. Credits, payments, and rows without a timestamp are excluded at import.

Work cost

Sum of Linkt amounts for all trips classified as work. This is what the company legitimately spent on work travel.

Personal cost

Sum of Linkt amounts for personal trips. Note: the Xero invoice uses the Charge amount (CAR rate if repriced), not this figure, so the invoice total may differ.

Unmapped tags

Count of distinct LPN values in the CSV that don't match any tag or rego in your driver list. Use Add unmapped tags from CSV in Drivers & Tags to add them.

Worked example

A single trip, step by step

Using a real row from your May 2026 statement:

08/05/2026 07:19 · Gateway Motorway — Murarrie North · LPN 962GS2 · −$9.30
1 Override? No override set. Continue.
2 Driver rule? LPN 962GS2 → Matt O'Conner. Default rule = Auto. Continue.
3 Route rule? Description contains Murarrie North → class = commute. Now check time: 07:19 is inside AM window 06:00–09:30. Day is Thursday (working day). Result: PERSONAL
✓ Rule shown in table: Route rule: Murarrie North (motorway — in commute window → personal)
Linkt amount: $9.30. If repriced to CAR rate: $5.78. Xero invoice charges $5.78 to Matt O'Conner.

Compare with a mid-day run on the same gantry:

06/05/2026 12:01 · Clem 7 — Clem7 Northbound · LPN 962GS2 · −$10.37
3 Route rule? Description contains Clem7. Looking at the default route table — Clem7 is not in the built-in routes, so no match. Continue to step 4.
4 Working hours fallback. Wednesday, 12:01. Wednesday is a working day. 12:01 is between 06:00–19:00. Result: WORK
✓ Rule shown in table: Within working hours. Not invoiced to driver. If you know Clem7 is actually a commute route, add it to Settings → Route rules as Clem7 → personal.
